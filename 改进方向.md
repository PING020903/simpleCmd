# simpleCmd v2.0：cmdTree 全面重构方案

## 零、弃旧立新

**完全弃用：**
- `CommandParseNode.c` — 链表式两级分发
- `CommandParseTable.c` — hash 表两级分发
- `cmdBind_register` / `cmdBind_findByHandler` / `RECORD_CMD_CALL` 体系

**全新实现：** `CommandParseTree.c` + `CommandParseTree.h`

**保留复用：** `ll.h`（动态模式遍历）、`cmd_MemoryAlloc`/`cmd_MemoryFree`（动态模式分配）

**需要改造：** `cmdUserStringParser.c`（`ParseSpace` 增加双引号 token 支持）

---

## 一、模式切换

```c
// CommandParseTree.h
#define CMDTREE_MODE_STATIC   1   // 静态数组模式（嵌入式）
#define CMDTREE_MODE_DYNAMIC  0   // 动态分配模式（桌面/资源充足）
```

两种模式共享完全相同的 API，零条件编译侵入调用方代码。

## 二、静态模式 — 数组 + 索引

### 2.1 配置宏

```c
#define CMDTREE_STATIC_MAX_NODES  32   // 最大节点数（含根节点）
#define CMDTREE_STATIC_MAX_DEPTH   6   // 最大路由深度
```

### 2.2 数据结构

```c
typedef short cmdTreeNodeRef;        // 节点引用 = 数组索引
#define CMDTREE_NULL   (-1)
#define CMDTREE_ROOT   (0)           // 0 号位固定为根节点

typedef struct {
    unsigned int        hash;          // 当前 token 的 FNV hash
    handler_fn_t        handler;       // NULL = 纯路由节点
    data_handler_fn_t   dataHandler;   // NULL = 无 BLE 数据回调
    cmdTreeNodeRef      parent;        // 父节点索引（root.parent = -1）
    cmdTreeNodeRef      first_child;   // 长子索引（-1 = 叶子）
    cmdTreeNodeRef      next_sibling;  // 兄弟索引（-1 = 末尾）
} cmdTreeNode_t;
```

### 2.3 内部存储

```c
static cmdTreeNode_t  cmdTree_nodes[CMDTREE_STATIC_MAX_NODES];
static short          cmdTree_nextFree = 1;   // 0 号已被根节点占用
```

### 2.4 注册逻辑

```c
cmdTreeNodeRef cmdTree_Register(cmdTreeNodeRef parent, const char *token, ...) {
    if (cmdTree_nextFree >= CMDTREE_STATIC_MAX_NODES)
        return CMDTREE_NULL;   // 表满

    short idx = cmdTree_nextFree++;
    cmdTreeNode_t *n = &cmdTree_nodes[idx];
    n->hash = fnv_hash(token);
    n->handler = handler;
    n->dataHandler = dataHandler;
    n->parent = parent;
    n->first_child = CMDTREE_NULL;
    n->next_sibling = CMDTREE_NULL;

    // 链入 parent 的 child 链表
    cmdTreeNode_t *p = &cmdTree_nodes[parent];
    if (p->first_child == CMDTREE_NULL) {
        p->first_child = idx;
    } else {
        short sib = p->first_child;
        while (cmdTree_nodes[sib].next_sibling != CMDTREE_NULL)
            sib = cmdTree_nodes[sib].next_sibling;
        cmdTree_nodes[sib].next_sibling = idx;
    }
    return idx;
}
```

### 2.5 子节点查找

```c
static cmdTreeNodeRef find_child(cmdTreeNodeRef parent, unsigned int hash) {
    short child = cmdTree_nodes[parent].first_child;
    while (child != CMDTREE_NULL) {
        if (cmdTree_nodes[child].hash == hash) return child;
        child = cmdTree_nodes[child].next_sibling;
    }
    return CMDTREE_NULL;
}
```

## 三、动态模式 — 外部内存分配 + c-linked-list

### 3.1 设计要点

- 每个节点通过 `cmd_MemoryAlloc` 分配，通过 `cmd_MemoryFree` 释放
- 子节点用 `ll_t` 双向链表组织，复用 `ll.h` 的 `list_for_each_entry` 遍历
- 每个节点内嵌一个 `ll_t` 作为自己在父节点 children 链表中的锚点，同时内嵌一个 `ll_t` 作为自己子节点链表的头

### 3.2 数据结构

```c
typedef struct cmdTreeNode {
    unsigned int            hash;
    handler_fn_t            handler;        // NULL = 纯路由节点
    data_handler_fn_t       dataHandler;    // NULL = 无 BLE 数据回调
    struct cmdTreeNode     *parent;         // 父节点指针（root->parent = NULL）

    ll_t                    sibling_node;   // 挂在 parent->children_head 链表上
    ll_t                    children_head;  // 本节点的子节点链表头
} cmdTreeNode_t;
```

### 3.3 内部存储

```c
static cmdTreeNode_t *cmdTree_root = NULL;   // 动态分配，作为根节点
// CMDTREE_ROOT 宏展开为 cmdTree_root
```

### 3.4 注册逻辑

```c
cmdTreeNode_t *cmdTree_Register(cmdTreeNode_t *parent, const char *token, ...) {
    cmdTreeNode_t *n = cmd_MemoryAlloc(sizeof(cmdTreeNode_t));
    if (!n) return NULL;

    n->hash = fnv_hash(token);
    n->handler = handler;
    n->dataHandler = dataHandler;
    n->parent = parent;
    n->children_head = (ll_t)ll_head_INIT(n->children_head);  // 初始化子链表

    // 链入 parent 的 children_head 链表
    list_add_tail(&n->sibling_node, &parent->children_head);

    return n;   // 返回的指针就是节点引用
}
```

### 3.5 子节点查找（利用 ll.h 遍历）

```c
static cmdTreeNode_t *find_child(cmdTreeNode_t *parent, unsigned int hash) {
    cmdTreeNode_t *child;
    list_for_each_entry(child, &parent->children_head, sibling_node) {
        if (child->hash == hash) return child;
    }
    return NULL;
}
```

### 3.6 深度限制

**无结构深度限制。** 节点由 `cmd_MemoryAlloc` 逐个分配，树可以无限套娃——10 层、20 层、100 层，只要内存分配不失败。

唯一的约束来自解析时的 `ParseSpace` 分词上限（`PARSE_SIZE`），即输入字符串长度。

### 3.7 释放

```c
void cmdTree_reset(void) {
    // 递归或迭代释放所有节点（先子后父）
    cmdTree_free_subtree(cmdTree_root);
    cmd_MemoryFree(cmdTree_root);
    cmdTree_root = NULL;
}
```

## 四、Token 分词器改造 — 双引号原始数据支持

### 4.1 需求

`ParseSpace` 当前按空格分割所有内容。但某些场景需要传递含空格的原始数据：

```
wait dat names "张三 李四 王五"
```

期望分词结果：`["wait", "dat", "names", "张三 李四 王五"]`（4 个 token），而非 6 个。

### 4.2 规则

- 英文双引号 `"` 包裹的内容视为 **一个 token**
- 引号本身不包含在 token 内容中（`strHead` 跳过 `"`，`len` 不含引号）
- 引号必须成对出现，未闭合的引号 → 从 `"` 到行尾作为一个 token
- 引号内可以包含空格、特殊字符
- 引号外仍按空格分割
- 空引号 `""` → 一个空 token（len=0）

### 4.3 分词示例

| 输入 | token 数 | token 内容 |
|------|---------|-----------|
| `cmd param` | 2 | `cmd`, `param` |
| `cmd "hello world"` | 2 | `cmd`, `hello world` |
| `"raw data" cmd param` | 3 | `raw data`, `cmd`, `param` |
| `cmd "a b" "c d"` | 3 | `cmd`, `a b`, `c d` |
| `cmd "unclosed string` | 2 | `cmd`, `unclosed string` |
| `cmd ""` | 2 | `cmd`, (空串) |
| `fatfs write "/path/with space/file.bin"` | 3 | `fatfs`, `write`, `/path/with space/file.bin` |

### 4.4 解析状态机

```
状态: NORMAL（普通分词，空格分隔）
      ↓ 遇到 "
状态: IN_QUOTE（引号内，空格不分割）
      ↓ 遇到 "
状态: NORMAL（引号闭合，回到普通分词）
      ↓ 到达字符串末尾
状态: END（未闭合的引号 → 从 " 到末尾作为一个 token）
```

### 4.5 对 tree 路由的影响

**无影响。** 引号 token 与普通 token 一样参与 FNV hash 计算和树路由。
对 tree 来说，`"张三 李四"` 就是一个 hash 值为 `fnv("张三 李四")` 的普通 token，行为与不带空格的 token 完全一致。

实际中引号 token 更多用于 **路由命中后作为 handler 的参数透传**：

```
输入: "wait dat names \"张三 李四\""
      → token[0]="wait" → 下行到 wait 节点
      → token[1]="dat"  → 下行到 dat 节点
      → token[2]="names" → 下行到 names 节点（命中 handler）
      → handler 拿到剩余参数: token[3]="张三 李四"
```

## 六、统一解析算法

```c
int cmdTree_CommandParse(const char *input) {
    ParseSpace(input);
    int tokenCnt = userParse_GetUserParamCnt();
    userString *tokens = userParse_pUserData();

    cmdTreeNodeRef cur = CMDTREE_ROOT;
    cmdTreeNodeRef bestMatch = CMDTREE_NULL;
    int bestDepth = -1;

    // 逐 token 沿树下行
#if CMDTREE_MODE_STATIC
    // 静态模式：编译期深度硬限制，防止越界
    const int maxDepth = CMDTREE_STATIC_MAX_DEPTH;
#else
    // 动态模式：无深度限制，纯靠树结构 & 内存决定
    const int maxDepth = tokenCnt;  // 最多走到 token 用完
#endif
    for (int i = 0; i < tokenCnt && i < maxDepth; i++) {
        unsigned int h = fnv_hash(tokens[i].strHead, tokens[i].len);
        cmdTreeNodeRef child = find_child(cur, h);
        if (child == CMDTREE_NULL) break;       // 下行失败

        cur = child;
        if (node(cur)->handler != NULL) {       // 记录最深命中
            bestMatch = cur;
            bestDepth = i;
        }
    }

    // 回退：当前节点有 handler 则以当前节点为 bestMatch
    if (bestMatch == CMDTREE_NULL && node(cur)->handler != NULL) {
        bestMatch = cur;
        bestDepth = /* 当前 token 位置 */;
    }

    if (bestMatch == CMDTREE_NULL) { lastError = ERR_NOT_FOUND; goto end; }

    // 自动设置 BLE 数据回调
    cmdTree_setActiveBinding(bestMatch);

    // 调用 handler，剩余参数透传
    cmdTreeNode_t *n = node(bestMatch);
    (bestDepth + 1 < tokenCnt)
        ? n->handler(&tokens[bestDepth + 1])
        : n->handler(NULL);

    lastError = OK;
end:
    RESET_USERDATA_RECORD();
    return lastError;
}
```

## 七、使用示例

```c
cmdTree_init();

// === 简单命令 ===
cmdTreeNodeRef dev = cmdTree_Register(CMDTREE_ROOT, "device", NULL, NULL);
cmdTree_Register(dev, "reset", cmdTask_deviceReset, NULL);
cmdTree_Register(dev, "ota",   cmdTask_deviceOta,   NULL);

// === 多级子系统 ===
cmdTreeNodeRef fatfs = cmdTree_Register(CMDTREE_ROOT, "fatfs", NULL, NULL);
cmdTreeNodeRef fcmd  = cmdTree_Register(fatfs, "cmd", NULL, NULL);
cmdTree_Register(fcmd, "mkdir",  fatfsMkdir,  NULL);
cmdTree_Register(fcmd, "ls",     fatfsLs,     NULL);
cmdTree_Register(fcmd, "format", fatfsFormat, NULL);

// === BLE 绑定：handler + dataHandler 一体 ===
cmdTreeNodeRef wait = cmdTree_Register(CMDTREE_ROOT, "wait", NULL, NULL);
cmdTreeNodeRef wdat = cmdTree_Register(wait, "dat", NULL, NULL);
cmdTree_Register(wdat, "names",  cmdTask_waitCnArea1, cmdTaskData_waitCnArea1);
cmdTree_Register(wdat, "desc",   cmdTask_waitCnArea2, cmdTaskData_waitCnArea2);
cmdTree_Register(wdat, "font",   cmdTask_waitFont,    cmdTaskData_waitFont);
cmdTree_Register(wdat, "jpeg",   cmdTask_waitPicture, cmdTaskData_waitPicture);

// === 中间节点可带 handler ===
cmdTreeNodeRef test = cmdTree_Register(CMDTREE_ROOT, "test", cmdTask_testHelp, NULL);
cmdTree_Register(test, "hardware", cmdTask_testHardware, NULL);
cmdTree_Register(test, "flash",    cmdTask_testFlash,    NULL);
// "test"           → cmdTask_testHelp(NULL)
// "test hardware"  → cmdTask_testHardware(NULL)
// "test hardware 0" → cmdTask_testHardware("0")
```

## 八、模式对照

| | 静态模式 | 动态模式 |
|---|---|---|
| 节点存储 | `cmdTreeNode_t nodes[MAX]` 定长数组 | 每个节点独立 `cmd_MemoryAlloc` |
| 节点引用 | `short`（数组索引） | `cmdTreeNode_t *`（指针） |
| 子节点组织 | `first_child` + `next_sibling`（short 链） | `children_head`（`ll_t` 双向链表） |
| 子节点遍历 | while 循环追踪索引 | `list_for_each_entry` 宏 |
| 树深度限制 | `CMDTREE_STATIC_MAX_DEPTH` 编译期硬限制 | **无限制**，纯靠内存分配是否成功。套 10 层、100 层均可 |
| 内存上限 | 编译期确定 `MAX_NODES` | 运行时由分配器决定 |
| 内存释放 | 无需（数组生命周期 = 程序） | `cmdTree_reset()` 递归释放 |
| 适用场景 | CH585M (18KB SRAM) | simpleCmd 桌面调试 / 资源充足 |
| 外部依赖 | 无 | `ll.h`、`cmd_MemoryAlloc`/`cmd_MemoryFree` |
