# 食用手册 — simpleCmd 项目搭建指南

> 本手册介绍如何从零搭建本项目的编译环境，以及如何通过命令行构建、运行测试。

---

## 一、你需要什么

| 工具 | 用途 | 最低版本 |
|------|------|----------|
| **LLVM MinGW** | C 编译器（i686-w64-mingw32-gcc） | `llvm-mingw-20220906-msvcrt-i686` |
| **CMake** | 工程构建配置 | 3.10 |
| **Ninja** | 快速并行构建工具 | 任意现代版本 |

> **为什么是 LLVM MinGW？** 本项目目标平台是嵌入式 MCU，开发环境用的是 LLVM MinGW 工具链。在桌面端用相同工具链编译测试，可以保持一致的 C 标准行为。

## 二、安装 LLVM MinGW

### 2.1 下载

从 GitHub Release 页面下载：

```
https://github.com/mstorsjo/llvm-mingw/releases
```

找到版本 `llvm-mingw-20220906-msvcrt-i686.zip`，下载并解压。

### 2.2 解压到固定路径

解压到 `F:\llvm-mingw-20220906-msvcrt-i686\`，使得目录结构为：

```
F:\llvm-mingw-20220906-msvcrt-i686\
├── bin\
│   ├── gcc.exe
│   ├── g++.exe
│   ├── llvm-ar.exe
│   └── ...
├── include\
└── lib\
```

> **关键**：项目中的[工具链文件](toolchain-llvm-mingw.cmake)写死了路径 `F:/llvm-mingw-20220906-msvcrt-i686`。如果你把 LLVM MinGW 装在其他位置，需要修改 `toolchain-llvm-mingw.cmake` 第 4 行的 `TOOLCHAIN_ROOT`。

### 2.3 添加到系统 PATH（重要）

`build.bat` 需要编译器在 PATH 中才能正常工作。将 LLVM MinGW 的 `bin` 目录添加到系统环境变量：

1. 打开「系统属性」→「环境变量」
2. 在「系统变量」中找到 `Path`，点击「编辑」
3. 新建一条：`F:\llvm-mingw-20220906-msvcrt-i686\bin`
4. 确定保存，**重新打开终端**使 PATH 生效

### 2.4 验证安装

打开 PowerShell，运行：

```powershell
gcc --version
```

如果成功，会输出类似：

```
gcc (llvm-mingw) ...  based on LLVM ...
```

## 三、安装 CMake（推荐 winget）

使用 Windows 包管理器 `winget` 一键安装，无需手动下载、勾选 PATH 等操作：

```powershell
winget install Kitware.CMake
```

安装完成后，**重新打开一个 PowerShell 窗口**，验证：

```powershell
cmake --version
```

输出应显示版本号 ≥ 3.10，例如 `cmake version 3.31.6`。

> 📖 详细图解可参考这篇 CSDN 教程：[cmake安装（Windows）](https://blog.csdn.net/qq_55745713/article/details/149883926?spm=1001.2014.3001.5501)（已亲测可行）。

## 四、安装 Ninja

### 4.1 下载

从 GitHub Release 页面下载：

```
https://github.com/ninja-build/ninja/releases
```

下载 `ninja-win.zip`，解压得到 `ninja.exe`。

### 4.2 放入 PATH

把 `ninja.exe` 放到一个在 PATH 中的目录，例如：

- `C:\Windows\` （最简单）
- 或者自己建一个目录（如 `C:\tools\`），然后添加到系统环境变量 PATH 中

### 4.3 验证安装

```powershell
ninja --version
```

输出版本号即可。

## 五、编译项目

### 5.1 打开项目目录

```powershell
cd F:\cmake_study\simpleCmd
```

### 5.2 一键构建（推荐）

项目提供了 [`build.bat`](build.bat)，双击运行或在终端中执行：

```powershell
.\build.bat
```

这个脚本会自动完成：

1. 清理旧进程
2. CMake 配置（Ninja + LLVM MinGW + Debug）
3. 并行编译
4. 启动主程序 `outputFile.exe`

### 5.3 手动构建（分步）

如果你想分步了解构建过程：

```powershell
# 第 1 步：CMake 配置（生成 build.ninja）
cmake -B build -G "Ninja" -DCMAKE_TOOLCHAIN_FILE="toolchain-llvm-mingw.cmake" -DCMAKE_EXPORT_COMPILE_COMMANDS=ON

# 第 2 步：编译
cmake --build build --parallel
```

编译产物在 `build\bin\` 目录下：

| 可执行文件 | 说明 |
|------------|------|
| `outputFile.exe` | 主程序（含 4 种 demo + REPL 交互） |
| `test_tokenizer.exe` | 分词器测试（13 项用例） |
| `test_cmdTree.exe` | cmdTree 命令解析器测试（11 项用例） |

## 六、运行测试

### 6.1 分词器测试

```powershell
.\build\bin\test_tokenizer.exe
```

输出示例：

```
=== Tokenizer Double-Quote Test Suite ===

[TEST 1] input: "cmd param"
  OK   token[0]: "cmd" (len=3)
  OK   token[1]: "param" (len=5)
  => PASS

...（共 13 项）...

=== Narrow-char Results: 13/13 passed ===
=== Total Results: 13/13 passed ===
```

测试覆盖：空格分隔、双引号包裹（含中文）、空引号、未闭合引号、宽字符（如果 `ENABLE_WCHAR=1`）等。

### 6.2 cmdTree 命令解析器测试

```powershell
.\build\bin\test_cmdTree.exe
```

输出示例：

```
[TEST 1] init / reset lifecycle
[TEST 2] simple two-level: device/reset
  ...（共 11 项）...

=== cmdTree Results: 11/11 passed ===
```

测试覆盖：初始化 / 重置、两级路由、命令未找到、多级路由（10 层深度）、引号参数透传、BLE 数据回调绑定、兄弟节点匹配、边界情况等。

### 6.3 主程序运行

```powershell
.\build\bin\outputFile.exe
```

会显示一个菜单，输入 1~4 选择不同的演示模式，其中 **3** 是交互式 REPL，可以输入命令测试。

## 七、常见问题

### Q: PowerShell 里执行 `gcc` 或 `cmake` 提示「无法识别」

A: 说明对应工具的 `bin` 目录没有加入 PATH。
- **gcc**：检查 LLVM MinGW 的 `bin` 目录是否已添加到系统环境变量（见 §2.3）
- **cmake**：如果用 winget 安装通常自动配好 PATH，重新打开终端即可；如果仍不行，手动将 CMake 安装目录下的 `bin` 添加到 PATH

### Q: 构建报 `The CMAKE_C_COMPILER is set to ... not found`

A: 可能原因有两个：
1. LLVM MinGW 的 `bin` 目录没加入 PATH（见 §2.3），编译器找不到
2. `toolchain-llvm-mingw.cmake` 第 4 行的 `TOOLCHAIN_ROOT` 没有指向实际的解压路径

### Q: 报 `ninja: error: ...`

A: 确认 `toolchain-llvm-mingw.cmake` 中的路径使用的是正斜杠 `/`（如 `F:/llvm-mingw-...`），不要用反斜杠 `\`。CMake 工具链文件中的路径需要使用正斜杠。

### Q: 测试输出中文乱码

A: 在 PowerShell 中先执行 `chcp 65001` 切换到 UTF-8 编码，或直接看英文输出结果（`PASS` / `FAIL` 关键词不受影响）。
